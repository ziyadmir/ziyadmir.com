Proof of Concept: Coding an MCP Client with Claude

July 15, 2025

Yesterday I put Claude Code Opus 4 through its paces by asking it to build a small repository from scratch: nba-player-stats-mcp. The goal was simple—answer questions like “How many rebounds did LeBron James get per game in 2023?” via a Model Context Protocol (MCP) client.

An AI-Driven Workflow

Every line of code in the repo came from Claude. My role was to steer with natural language prompts: choose libraries, debug when they failed, and provide context via documentation links. The entire session ran inside windsurf at the max+opus 4 setting, giving me a conversational workflow right in the terminal.

Coding from the Terminal

I’ve always favored a keyboard-driven environment—think tmux and vim. Windsurf lets me maintain that flow state while collaborating with AI. Claude handled boilerplate and API calls; I focused on clarifying intent. In about three hours of intermittent work, we had a fully functional client that fetches player stats.

Connecting to the Agent Platform

This lightweight project mirrors our larger vision with our agent orchestration platform, our step-based agent platform. The same run-loop concept applies: keep all context in one place and let agents orchestrate the tasks. The repo proved that even a single-agent workflow can deliver results with minimal human involvement.

Why It Matters

The manual pieces—finding libraries, validating output—are all solvable within the MCP framework. Once integrated, an agent can pull code, docs, and data sources from across the internet. The terminal becomes a universal interface for AI-first development.

It’s a small proof of concept, but it hints at a larger shift. With AI assistants embedded in our tooling, we can iterate on ideas in real time and keep every decision in version control. The symmetries with our agent platform are clear: isolate steps, preserve context, and let machines handle the busy work.
