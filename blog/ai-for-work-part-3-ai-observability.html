The Hidden Gold Mine

Your security team already mandates that all outbound traffic goes through a corporate proxy. What if I told you this "compliance burden" is actually your most valuable AI analytics asset?

Every LLM request from your organization—every prompt, every response, every context injection—flows through that proxy. At scale, this becomes a real-time stream of how AI is transforming your engineering culture.

We built an AI-driven analysis system on top of our proxy logs. It processes 15-20M events daily, provides sub-minute security alerting, and delivers insights that have fundamentally changed how we think about AI adoption.

The Data Deluge Problem

When we first enabled proxy logging for LLM traffic, we were overwhelmed:

Volume: 20GB of JSON logs per day Velocity: 200 requests per second at peak Variety: IDEs, CLI tools, browser extensions, custom scripts Veracity: How do you separate signal from noise?

Traditional approaches failed:

  • SIEM rules couldn't handle the variety
  • Dashboards showed vanity metrics
  • Manual sampling missed critical events
  • Storage costs exploded

We needed something smarter.

Architecture: AI Analyzing AI

The solution? Use AI to analyze AI usage. Here's our architecture:

Stream Processing Pipeline

1. Ingestion Layer

First, we capture and normalize the proxy logs:

`python class ProxyLogProcessor: def process_log_entry(self, raw_log: dict) -> StandardizedEvent: # Extract LLM request/response from proxy payload if self.is_llm_traffic(raw_log): return StandardizedEvent( timestamp=raw_log['timestamp'], user_id=self.extract_user(raw_log), endpoint=raw_log['url'], method=raw_log['method'], request_payload=self.decode_payload(raw_log['request_body']), response_payload=self.decode_payload(raw_log['response_body']), latency_ms=raw_log['response_time'], status_code=raw_log['status_code'] ) def is_llm_traffic(self, log: dict) -> bool: llm_domains = [ 'api.openai.com', 'api.anthropic.com', 'generativelanguage.googleapis.com', 'api.cohere.ai', 'your-internal-llm.company.com' ] return any(domain in log['url'] for domain in llm_domains) `

2. Real-Time Analysis Agents

We run specialized AI agents on the event stream:

Security Agent `python class SecurityAnalysisAgent: def __init__(self): self.secret_patterns = self.load_secret_patterns() self.anomaly_detector = self.load_anomaly_model() async def analyze(self, event: StandardizedEvent) -> SecurityAnalysis: results = await asyncio.gather( self.check_secret_leakage(event), self.detect_jailbreak_attempts(event), self.identify_policy_violations(event), self.detect_anomalous_behavior(event) ) return SecurityAnalysis( risk_score=self.calculate_risk_score(results), findings=results, recommended_actions=self.determine_actions(results) ) async def check_secret_leakage(self, event: StandardizedEvent) -> Finding: prompt = event.request_payload.get('messages', []) # Use local LLM for speed analysis = await self.local_llm.analyze({ "task": "identify_secrets", "content": prompt, "patterns": self.secret_patterns }) if analysis.found_secrets: return Finding( severity="CRITICAL", type="SECRET_LEAK", details=analysis.details, evidence=analysis.matched_content ) `

Productivity Agent `python class ProductivityAnalysisAgent: async def analyze(self, event: StandardizedEvent) -> ProductivityMetrics: # Extract metrics from the interaction metrics = { 'tokens_consumed': self.count_tokens(event), 'response_accepted': self.was_accepted(event), 'task_category': self.categorize_task(event), 'quality_score': self.assess_quality(event) } # Aggregate to user/team level await self.update_aggregates(event.user_id, metrics) return ProductivityMetrics( user_efficiency=self.calculate_efficiency(metrics), task_complexity=self.assess_complexity(event), time_saved_estimate=self.estimate_time_saved(metrics) ) def categorize_task(self, event: StandardizedEvent) -> str: # Use embeddings to classify the type of work embedding = self.embed_prompt(event.request_payload) return self.task_classifier.predict(embedding) # Returns: 'code_generation', 'debugging', 'documentation', etc. `

3. Action Layer

Based on analysis, we trigger automated responses:

`python class ActionOrchestrator: async def handle_findings(self, analysis: AnalysisResult): if analysis.security.risk_score > 0.8: await self.high_risk_response(analysis) elif analysis.security.risk_score > 0.5: await self.medium_risk_response(analysis) # Always record metrics await self.record_metrics(analysis) async def high_risk_response(self, analysis: AnalysisResult): # Immediate actions for critical issues await asyncio.gather( self.send_slack_alert( channel="#security-incidents", message=self.format_security_alert(analysis) ), self.create_jira_ticket( project="SECURITY", priority="P1", details=analysis.security.findings ), self.notify_user( user_id=analysis.event.user_id, message="Potential security issue detected" ) ) # Optional: Rate limit the user if "SECRET_LEAK" in analysis.security.finding_types: await self.rate_limiter.throttle_user( analysis.event.user_id, duration_minutes=30 ) `

Deep Dive: Detection Patterns

Secret Leakage Detection

We use a multi-layered approach:

`python class SecretDetector: def __init__(self): # Layer 1: Regex patterns self.patterns = { 'aws_key': r'AKIA[0-9A-Z]{16}', 'github_token': r'gh[ps]_[a-zA-Z0-9]{36}', 'private_key': r'-----BEGIN (?:RSA |EC )?PRIVATE KEY-----', 'jwt': r'eyJ[a-zA-Z0-9_-]\.eyJ[a-zA-Z0-9_-]\.[a-zA-Z0-9_-]*' } # Layer 2: Entropy analysis self.entropy_threshold = 4.5 # Layer 3: Context-aware LLM self.llm_analyzer = self.load_secret_detection_model() async def detect(self, content: str) -> List[Secret]: secrets = [] # Quick regex scan for name, pattern in self.patterns.items(): if match := re.search(pattern, content): secrets.append(Secret(type=name, value=match.group())) # Entropy-based detection for unknown secrets for token in self.tokenize(content): if self.calculate_entropy(token) > self.entropy_threshold: # Use LLM to verify if it's actually a secret if await self.llm_analyzer.is_secret(token, content): secrets.append(Secret(type='high_entropy', value=token)) return secrets `

Jailbreak Detection

Identifying attempts to bypass AI safety measures:

`python class JailbreakDetector: def __init__(self): self.known_techniques = self.load_jailbreak_database() self.embedding_model = self.load_embedding_model() async def detect_jailbreak(self, prompt: str) -> JailbreakAnalysis: # Check against known patterns for technique in self.known_techniques: if technique.matches(prompt): return JailbreakAnalysis( detected=True, technique=technique.name, confidence=technique.confidence ) # Semantic similarity to known jailbreaks prompt_embedding = self.embedding_model.encode(prompt) similarities = self.calculate_similarities( prompt_embedding, self.jailbreak_embeddings ) if max(similarities) > 0.85: return JailbreakAnalysis( detected=True, technique='semantic_similarity', confidence=max(similarities) ) # Behavioral analysis if self.looks_suspicious(prompt): return JailbreakAnalysis( detected=True, technique='behavioral', confidence=0.7 ) `

Anomaly Detection

Catching unusual usage patterns:

`python class AnomalyDetector: def __init__(self): self.user_profiles = {} self.isolation_forest = IsolationForest(contamination=0.1) def detect_anomalies(self, event: StandardizedEvent) -> AnomalyScore: user_id = event.user_id # Build feature vector features = np.array([ event.tokens_consumed, event.latency_ms, self.get_hour_of_day(event.timestamp), self.get_request_complexity(event), self.get_endpoint_rarity(event.endpoint) ]) # Update user profile if user_id not in self.user_profiles: self.user_profiles[user_id] = UserProfile() profile = self.user_profiles[user_id] anomaly_score = profile.calculate_anomaly_score(features) # Global anomaly detection global_anomaly = self.isolation_forest.decision_function([features])[0] return AnomalyScore( user_anomaly=anomaly_score, global_anomaly=global_anomaly, is_anomalous=anomaly_score > 3.0 or global_anomaly < -0.5 ) `

Understanding AI's impact on development:

`sql -- Daily active AI users SELECT DATE(timestamp) as date, COUNT(DISTINCT user_id) as daily_active_users, SUM(tokens_consumed) as total_tokens, AVG(quality_score) as avg_quality FROM ai_usage_events GROUP BY DATE(timestamp);

-- Task distribution SELECT task_category, COUNT(*) as request_count, AVG(time_saved_seconds) as avg_time_saved, SUM(time_saved_seconds) / 3600 as total_hours_saved FROM ai_productivity_metrics WHERE timestamp > NOW() - INTERVAL '7 days' GROUP BY task_category ORDER BY request_count DESC;

-- Team adoption metrics SELECT team_name, COUNT(DISTINCT user_id) as active_users, COUNT(*) / COUNT(DISTINCT user_id) as avg_requests_per_user, AVG(acceptance_rate) as avg_acceptance_rate FROM ai_usage_events e JOIN users u ON e.user_id = u.id GROUP BY team_name ORDER BY active_users DESC; `

Security and Privacy Considerations

Data Minimization

We hash sensitive fields before storage:

`python def sanitize_event(event: StandardizedEvent) -> SanitizedEvent: return SanitizedEvent( timestamp=event.timestamp, user_hash=hashlib.sha256(event.user_id.encode()).hexdigest(), endpoint=event.endpoint, tokens_consumed=event.tokens_consumed, # Don't store actual prompts long-term prompt_hash=hashlib.sha256(event.request_payload.encode()).hexdigest(), task_category=event.task_category, risk_indicators=event.security_findings ) `

Access Controls

Strict access to analysis results:

`python class AccessControl: def can_view_security_data(self, user: User, data: SecurityData) -> bool: # Security team can see everything if user.role == 'security_analyst': return True # Managers can see their team's aggregated data if user.role == 'manager' and data.level == 'team': return data.team_id in user.managed_teams # Users can only see their own data if data.level == 'user': return data.user_id == user.id return False ``
