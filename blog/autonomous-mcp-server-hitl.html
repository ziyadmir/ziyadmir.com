Building an Autonomous MCP Server with HITL Guidance

July 2, 2025

Recently I set out to see if a production-grade Model Context Protocol (MCP) server could be written entirely by an AI assistant. The result is this working server that answers basketball questions on demand.

Autonomous Coding with Claude

Every line of code came from Claude Code Opus 4. I started with an empty repository and guided the model using natural language. Claude generated the server scaffolding, wrote the scraping logic for Basketball Reference, and even set up automated tests.

Human-in-the-Loop Steering

I only intervened when the model hit an error or got stuck. My prompts were simple—tell it what to test, ask for evidence that the tests passed, and push it to debug failing cases. That HITL approach ensured quality while keeping the workflow fully AI-driven.

Custom Basketball Stats

The finished server can answer questions like “What was Steph Curry’s three-point percentage in 2022?” by fetching live data from Basketball Reference. It’s a tiny example, but it demonstrates how an AI-generated MCP server can serve real requests with minimal human effort.

Why It Matters

This project proved that an AI can autonomously build a functioning backend with human oversight. The same pattern scales to far more complex systems—agents do the busy work while humans guide the tests and architecture. It’s a glimpse of how software development will work in the AI-first era.
