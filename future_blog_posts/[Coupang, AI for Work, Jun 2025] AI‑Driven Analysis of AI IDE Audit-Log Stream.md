## **Author**: Ziyad Mir, Jun 2025

### **Background**

*Read this first\! [\[Coupang, AI for Work, Jun 2025\] Replacing MCP Audit Logging with a Zscaler Proxy-Based Solution](https://docs.google.com/document/d/1vLHOYU-xOKaoW4hUdivNlWH6zJuSQsGqm__3XrCZoac/edit?tab=t.0)*

[Coupang’s Zscaler proxy will soon capture every outbound LLM request/response generated by the AI IDE and other GenAI tools](https://docs.google.com/document/d/1vLHOYU-xOKaoW4hUdivNlWH6zJuSQsGqm__3XrCZoac/edit?tab=t.0), producing a high‑volume JSON log stream (≈ 15‑20 M events/day). Today, no system continuously reviews this data for policy violations **or** leverages it to understand how GenAI is accelerating developer workflows. The raw logs live in Zscaler Nanolog / S3 with limited search‑and‑export capability.

---

### **Motivation**

| Pain‑Point | Impact |
| ----- | ----- |
| **Security Blind Spots** – Manual sampling cannot catch prompt leaks, jailbreak attempts, or anomalous usage patterns in near‑real time. | Compliance risk & delayed incident response. |
| **Data Deluge** –  TB‑scale daily logs exceed what traditional dashboards or SIEM rules can triage. | Analysts overwhelmed; insights lost. |
| **Separate Tooling for Productivity Metrics** – AIIDE maintains its own analytics code, duplicating ingest, storage, and dashboards. | Engineering overhead & inconsistent truth. |

A unified, intelligent analyser eliminates duplicate pipelines, surfaces actionable security incidents, and distils company‑wide productivity signals from the same data source.

---

### **Proposed Solution Design**

| Layer | What it Does | Key Tech |
| ----- | ----- | ----- |
| **1\. Stream Ingest** | Nanolog → Amazon Kinesis/S3 → Kafka topic `ai-logs` (flattened, PII‑hashed). | Zscaler NSS or S3 sync, AWS Firehose, Flink ETL. |
| **2\. Real‑Time AI Agents** | EMUX workflows classify each event for:  • **Security** – secret leakage, policy mismatch, jailbreak patterns, behavioural anomalies.  • **Productivity** – latency, token volume, prompt‑quality scoring, usage clustering. | EMUX step runner, lightweight ML classifiers, local LLM (Mixtral‑8x7B) with RAG. |
| **3\. Action & Visibility** | • Slack / Jira alerts for high‑risk events. • Prometheus \+ Grafana dashboards for latency & adoption trends. • Optional: MCP rate‑limit API for auto‑remediation. | Webhooks, Prom‑pushgateway, Grafana, MCP admin API. |
| **4\. Data Lake & Feature Store** | Raw parquet (1 yr), curated feature tables (30 d), vector index (90 d). | S3, Glue/Athena, OpenSearch Vector. |

End‑to‑end latency: \< 30 s from proxy event to alert or metric.

---

### **Ownership & Collaboration**

* **Engineering Efficiency (AI Platform / EMUX)** – Builds and operates the Flink ingest job, EMUX agents, Grafana boards; deprecates legacy AIIDE analytics code.

* **Security (InfoSec / SOC)** – Defines detection policies, monitors Slack/Jira queue, owns incident playbooks.

* **Data Platform** – Provides Kafka, S3 lake, and Prometheus infrastructure; supports cost optimisation.

* **Zscaler** – Ensures log completeness, assists with advanced DLP matchers, tunes NSS throughput.

---

### **Benefits**

| Benefit | Detail |
| ----- | ----- |
| **Full‑Spectrum Security Coverage** | 100 % of LLM traffic scanned; \< 1 min alerting SLA; replayable evidence for audits. |
| **Zero‑Friction Developer Experience** | Analysis is asynchronous; no IDE latency hit, no MCP code paths touched. |
| **One Pipeline, Two Outcomes** | Same event stream powers compliance **and** productivity KPIs—eliminates duplicate instrumentation. |
| **Auditable & Extensible** | EMUX stores step‑level inputs/outputs; new detectors or metrics drop in as additional steps. |
| **Cost‑Efficient** | Serverless ingest \+ autoscaled K8s workers; \~$500/month at projected 10 TB/month volume. |

---

### **Summary**

By attaching an **AI‑driven analysis layer** to the Zscaler LLM log stream, Coupang gains continuous security monitoring ***and*** actionable insight into GenAI adoption—all without impacting developer workflows or multiplying analytics stacks. The single EMUX‑based system turns raw proxy logs into real‑time alerts, rich dashboards, and data‑backed optimisation levers, delivering a clear, methodical win for both compliance and engineering velocity.
